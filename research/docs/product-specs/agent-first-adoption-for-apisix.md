# Agent-First Adoption for APISIX

## 문제 정의

Codex 같은 에이전트가 APISIX 저장소에서 높은 생산성을 내려면, 광범위한 공식 문서와 테스트 자산을 짧은 탐색 경로로 압축해야 한다.

## 도입 가설

다음 조건이 충족되면 APISIX는 agent-first 연구 및 유지보수에 매우 적합하다.

- 명확한 진입점 문서 존재
- 설정 리소스 모델의 요약 존재
- 테스트 분류 인덱스 존재
- 관측 가능성 신호와 운영 시나리오의 연결 문서 존재
- 플러그인 패밀리별 요약 존재

## 기대 효과

- 신규 에이전트가 더 빠르게 문맥을 확보함
- 특정 버그 재현 경로를 테스트로 더 빨리 연결함
- 운영 모드별 위험 차이를 더 쉽게 설명함
- AI Gateway 및 MCP 관련 기능을 전략적으로 탐색 가능함

## 주요 리스크

- 지식 중복이 늘어나면 연구 문서가 빠르게 낡을 수 있음
- 코드, 공식 문서, 연구 문서 간 드리프트가 발생할 수 있음
- agent-first 문서가 너무 커지면 오히려 탐색 비용이 증가함

## 권장 원칙

- 긴 설명보다 인덱스와 교차 링크를 우선한다.
- 개념 설명과 사실 근거를 분리한다.
- 실행 가능한 예시는 가능한 테스트나 공식 설정 예제를 가리키게 한다.
