# Active Research Plan

## 목표

원문의 agent-first 문서 구조를 참고해 APISIX 저장소를 재해석하고, Codex 활용 가능성이 높은 조사 지점을 체계화한다.

## 단계

1. 저장소의 공식 진입점 파악
2. 런타임 코어와 플러그인 계층 분해
3. 배포 모드와 구성 모델 정리
4. 테스트와 CI를 품질 게이트 관점에서 정리
5. 관측 가능성과 운영 피드백 루프 정리
6. agent-first 도입 시 추가 문서화가 필요한 부분 식별

## 현재 확인된 핵심 근거

- README가 제품 범위와 최신 AI Gateway 포지션을 설명함
- architecture docs가 코어와 플러그인 관계를 요약함
- deployment docs가 운영 모델 차이를 명시함
- testing framework docs가 검증 방식의 진입점을 제공함
- config example이 런타임과 운영성을 세밀하게 드러냄

## 산출물

- 상위 맵 문서
- 설계 신념 문서
- 제품 관점 문서
- 신뢰성, 보안, 품질 문서
- 참고용 에이전트 읽기 자료
