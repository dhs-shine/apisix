# Generated Schema Placeholder

APISIX는 전형적인 애플리케이션 데이터베이스 스키마 중심 시스템이 아니다. 이 저장소에서 더 중요한 것은 관계형 DB 스키마보다 다음과 같은 구성 리소스 스키마다.

- routes
- upstreams
- services
- consumers
- consumer groups
- ssl resources
- plugin configs
- plugin metadata
- global rules
- secrets

향후 연구에서는 관계형 DB 스키마 대신 이 리소스 모델의 필드 구조와 상호 참조를 추출한 `resource-schema.md` 형태의 문서가 더 적합하다.
