# APISIX Agent-First Research Map

## 목적

이 디렉토리는 Apache APISIX 코드베이스를 에이전트 우선 엔지니어링 관점에서 분석한 연구 저장소다. 구현 변경을 제안하거나 직접 수정하기보다, 현재 저장소가 이미 제공하는 구조적 신호를 정리하고, Codex 같은 에이전트가 어디에서 높은 레버리지를 얻을 수 있는지 문서화하는 데 집중한다.

## 읽기 순서

1. `ARCHITECTURE.md`
2. `docs/PLANS.md`
3. `docs/design-docs/index.md`
4. `docs/product-specs/index.md`
5. `docs/QUALITY_SCORE.md`
6. `docs/RELIABILITY.md`
7. `docs/SECURITY.md`

## 연구 원칙

- 원 저장소의 실제 파일과 문서를 근거로 판단한다.
- 구현 지시보다 관찰, 구조화, 위험 식별을 우선한다.
- 에이전트가 읽기 쉬운 형태로 도메인 지식과 운영 지식을 분리한다.
- APISIX를 단순 API 게이트웨이가 아니라 운영 가능한 플랫폼으로 다룬다.

## 연구 범위

- 코어 런타임과 요청 처리 흐름
- 플러그인 시스템과 확장성
- 배포 모드와 운영 모델
- 테스트, CI, 품질 게이트
- 관측 가능성, 안정성, 보안 신호
- 에이전트 친화적 문서화 가능성

## 비범위

- 코드 수정
- 설정 변경
- 새로운 테스트 작성
- 성능 벤치마크 재실행

## 문서 해석 규칙

- `design-docs/`는 설계 원칙과 구조적 통찰을 담는다.
- `product-specs/`는 사용자 가치와 도입 시나리오를 담는다.
- `exec-plans/`는 조사 관점의 실행 계획과 후속 연구 큐를 담는다.
- `references/`는 에이전트가 빠르게 재사용할 수 있는 압축 지식을 담는다.
